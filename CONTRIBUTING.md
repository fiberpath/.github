# Contributing to FiberPath

Thanks for your interest in contributing! This is the organization-wide default;
individual repositories may add their own `CONTRIBUTING` with project-specific
detail, which takes precedence.

## Before you start

- Search existing issues and pull requests first.
- For substantial changes, open an issue to discuss the approach before writing
  code.

## Conventions across the org

- **Commits:** [Conventional Commits](https://www.conventionalcommits.org/) —
  `type(scope): description` (`feat`, `fix`, `docs`, `refactor`, `test`, `ci`,
  `chore`, `build`), imperative mood, one logical change per commit.
- **Branches:** work on a feature branch; the default branch is protected.
- **Pull requests:** keep them focused; ensure CI passes; reference the issue
  they address.
- **Checks:** run the repository's lint/format/test before pushing; do not
  bypass pre-commit hooks or skip CI.

## Project-specific guides

- **`fiberpath`** (core software): see the
  [Contributing guide](https://fiberpath.org/fiberpath/development/contributing/)
  and [Tooling](https://fiberpath.org/fiberpath/development/tooling/) docs.

By contributing, you agree that your contributions are licensed under the same
license as the repository you are contributing to.
